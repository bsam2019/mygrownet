### Integration Model: GrowFinance as a CMS Module

**GrowFinance is enabled just like other CMS modules:**

```
CMS Settings → Modules
├── ☑ Project Management
├── ☑ Inventory
├── ☑ Fleet Management
├── ☑ HR & Payroll
├── ☑ Construction Modules
└── ☑ GrowFinance (Full Accounting) ← NEW MODULE (IMPLEMENTED!)
```

**When enabled:**
- Auto-creates sync configuration
- Sets up default account mappings
- All CMS transactions sync automatically
- CMS gets enhanced financial reports

**Users don't see GrowFinance** - They just use CMS as normal and get complete accounting features.

---

### Integration Architecture: Event-Driven Sync

**Core Principle:** Avoid duplication by making GrowFinance the **accounting layer** while CMS remains the **operational layer**.

```
┌─────────────────────────────────────────────────────────┐
│                    CMS (Operations)                      │
│  - Invoices, Expenses, Payments (operational data)      │
│  - Job costing, budgets, P&L (operational reports)      │
└─────────────────┬───────────────────────────────────────┘
                  │ Events (async)
                  ↓
┌─────────────────────────────────────────────────────────┐
│              GrowFinance (Accounting)                    │
│  - Journal entries, chart of accounts                   │
│  - General ledger, balance sheet, cash flow             │
│  - Bank reconciliation, tax reports                     │
└─────────────────────────────────────────────────────────┘
```

**Key Design Decisions:**

1. **No Duplication**: CMS doesn't store journal entries, GrowFinance doesn't store invoices
2. **Single Source of Truth**: CMS owns operational data, GrowFinance owns accounting data
3. **Async Sync**: Events trigger queue jobs (no blocking operations)
4. **Idempotent**: Can retry failed syncs without creating duplicates
5. **Reversible**: Can disable integration without data loss

### Phase 2: Sync Logic (Week 2)

#### 2.1 Invoice Sync Handler

**Logic:**

```php
class InvoiceSyncHandler
{
    public function sync(InvoiceModel $invoice): void
    {
        // 1. Idempotency check
        if ($invoice->growfinance_synced) {
            return; // Already synced, skip
        }

        // 2. Get account mappings
        $salesAccount = $this->mappingService->getAccount(
            $invoice->company_id, 'invoice', 'sales'
        );
        $vatAccount = $this->mappingService->getAccount(
            $invoice->company_id, 'invoice', 'vat'
        );
        
        // 3. Build journal entry (double-entry accounting)
        $lines = [];
        
        // Debit: Accounts Receivable (or Cash if paid)
        if ($invoice->status === 'paid') {
            $cashAccount = $this->mappingService->getCashAccount(
                $invoice->company_id, 
                $invoice->payment_method
            );
            $lines[] = [
                'account_id' => $cashAccount->id,
                'debit_amount' => $invoice->total_amount,
                'credit_amount' => 0,
            ];
        } else {
            $receivableAccount = $this->mappingService->getAccount(
                $invoice->company_id, 'invoice', 'receivable'
            );
            $lines[] = [
                'account_id' => $receivableAccount->id,
                'debit_amount' => $invoice->total_amount,
                'credit_amount' => 0,
            ];
        }
        
        // Credit: Sales Revenue
        $lines[] = [
            'account_id' => $salesAccount->id,
            'debit_amount' => 0,
            'credit_amount' => $invoice->net_amount,
        ];
        
        // Credit: VAT Payable
        if ($invoice->vat_amount > 0) {
            $lines[] = [
                'account_id' => $vatAccount->id,
                'debit_amount' => 0,
                'credit_amount' => $invoice->vat_amount,
            ];
        }
        
        // 4. Create journal entry in GrowFinance
        DB::transaction(function () use ($invoice, $lines) {
            $journalEntry = $this->accountingService->createJournalEntry(
                businessId: $invoice->company_id,
                description: "Invoice #{$invoice->invoice_number}",
                lines: $lines,
                reference: "CMS-INV-{$invoice->id}"
            );
            
            // Post the entry (updates account balances)
            $this->accountingService->postJournalEntry($journalEntry);
            
            // Update sync status
            $invoice->update([
                'growfinance_synced' => true,
                'growfinance_journal_entry_id' => $journalEntry->id
            ]);
            
            // Log success
            $this->syncStatusService->logSuccess($invoice, $journalEntry);
        });
    }
}
```

**Prevents Duplication:**
- ✅ Checks `growfinance_synced` flag before syncing
- ✅ Uses unique reference `CMS-INV-{id}`
- ✅ Transaction wrapping ensures atomicity

### Avoiding Duplication: Key Strategies

#### 1. **Single Source of Truth**

```
CMS owns:                    GrowFinance owns:

- Expenses                   - Chart of accounts
- Payments                   - Account balances
- Job costing                - General ledger
- Budgets                    - Financial statements
- P&L (operational)          - Balance sheet
                             - Cash flow statement
```

#### 2. **Idempotency**

```php
// Before syncing, always check
if ($invoice->growfinance_synced) {
    return; // Skip, already synced
}

// Use unique references
$reference = "CMS-INV-{$invoice->id}"; // Unique per invoice

// Check if journal entry exists
$existing = GrowFinanceJournalEntry::where('reference', $reference)->first();
if ($existing) {
    return; // Skip, already exists
}
```

#### 3. **Transaction Wrapping**

```php
DB::transaction(function () {

    // All or nothing - if any step fails, rollback everything
```

### Avoiding Instability: Key Strategies

#### 1. **Async Processing**

```php
// Don't sync immediately (blocks CMS)
// Instead, dispatch job
SyncInvoiceToGrowFinanceJob::dispatch($invoice)
    ->onQueue('growfinance-sync')  // Separate queue
    ->delay(now()->addSeconds(5));  // Small delay
```

**Benefits:**
- ✅ CMS operations not blocked
- ✅ Can handle sync failures without affecting CMS
- ✅ Can retry failed syncs later

#### 2. **Graceful Failure Handling**

```php
try {
    $this->syncInvoice($invoice);
} catch (\Exception $e) {
    // Log error
    Log::error('GrowFinance sync failed', [
        'invoice_id' => $invoice->id,
        'error' => $e->getMessage()
    ]);
    
    // Update sync status
    $this->syncLog->create([
        'sync_status' => 'failed',
        'error_message' => $e->getMessage(),
        'sync_attempt_count' => $attempts + 1,
    ]);
    
    // Alert admin if too many failures
    if ($this->getFailureRate() > 0.1) { // 10% failure rate
        $this->alertAdmin();
    }
    
    // Don't throw - let CMS continue working
}
```

#### 3. **Circuit Breaker Pattern**

```php
class GrowFinanceSyncService
{
    public function sync($entity): void
    {
        // If too many recent failures, stop syncing temporarily
        if ($this->isCircuitOpen()) {
            Log::warning('GrowFinance sync circuit open, skipping');
            return;
        }
        
        try {
            $this->performSync($entity);
            $this->recordSuccess();
        } catch (\Exception $e) {
            $this->recordFailure();
            
            // Open circuit if failure threshold reached
            if ($this->shouldOpenCircuit()) {
                $this->openCircuit();
                $this->alertAdmin();
            }
        }
    }
}
```

**Benefits:**
- ✅ Prevents cascade failures
- ✅ Gives system time to recover
- ✅ Alerts admin to investigate

#### 4. **Monitoring & Alerts**
